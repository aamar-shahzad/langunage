Under the hood

Three models. One pipeline.

Your voice travels through three AI models, all running in WebAssembly in your browser tab. Here's exactly what happens in each step.

① Hearing — Speech to Text
Xenova/whisper-tiny

Your microphone audio is captured as a Float32Array, resampled to 16kHz, and fed into Whisper. The model uses an encoder-decoder transformer to output a token sequence which is decoded to text. WebGPU acceleration is used automatically when available.

② Thinking — Language Model
onnx-community/Qwen2.5-0.5B-Instruct

The transcript is formatted as a chat prompt with a system message defining the coach persona, target language, and correction mode. Qwen2.5 generates the response token-by-token. The response includes a correction markup tag that the UI parses to highlight errors.

③ Speaking — Text to Speech
Xenova/speecht5_tts

The clean response text is passed to SpeechT5, a fine-tuned TTS model. A speaker embedding from the CMU ARCTIC dataset shapes the voice character. The output is a Float32Array of PCM samples played back via the Web Audio API.
